CATEGORIZATION MODEL: FLAT / single-parent.
  Each resource row has ONE `category` text value that must equal a `categories.name`.
  A resource is counted exactly once under its category (and once under its sub /
  sub-sub node). No join/tag table — no resource appears under two categories.

~370 DISCREPANCY CAUSE (the old "total: 2210" vs 1838):
  - 372 rows were URL-duplicates: multiple approved rows sharing the same
    normalized (lower/trim) url. These inflated the raw resources table count.
  - 1 row was a category-orphan (id 184991 "QUANTEEC"): its `category` text did
    not match any categories.name, so it was invisible to /api/categories.
  - 6 rows carried tracking/shortener URLs (data-quality, not a count issue).

REMEDIATION (Option A — clean the data; all via admin API, journaled):
  - 372 URL-duplicate rows deleted (0 failures; deleted.json records id->200 for all 372).
  - Orphan 184991 reassigned -> Infrastructure & Delivery / Peer-to-Peer Streaming Solutions.
  - 6 links rewritten to direct URLs.
  NOTE ON REVERSIBILITY: the deletes are HARD deletes. The journal stored only ids +
  HTTP status, not full row contents, so a delete is NOT trivially re-insertable from
  the journal. Recovery paths that DO exist: Replit's automatic DB checkpoints
  (platform-level rollback) and re-adding any resource via the admin panel (URL is
  re-enrichable). The reassignment and link rewrites ARE reversible via the admin UI.
