## Reference renderer

The handoff ships two self-contained HTML bundles at
`/tmp/handoff/awesome-list-site/project/`:

- `Awesome.Video - Standalone.html` — **multi-page React demo** (use this).
  Root component is `app.jsx`; routes are React state, not URLs (see "Routing
  caveat" below).
- `index.standalone.html` — single-thumbnail showcase splash. Not useful for
  page-level diffs.

The bundle uses inline `__bundler/manifest` + `DecompressionStream` and works
fine over `file://` in a Chromium-based agent-browser session. Opening it
takes ~6–8s to unpack before the React tree mounts.

**How to open in agent-browser** (single shell so the daemon session
persists):

```bash
npx agent-browser open "file:///tmp/handoff/awesome-list-site/project/Awesome.Video%20-%20Standalone.html" --viewport 1280x800 \
  && npx agent-browser wait --load networkidle \
  && sleep 8 \
  && npx agent-browser screenshot --full --screenshot-format jpeg --screenshot-quality 85 screenshots/reference/NN_name.jpg
```

A smoke screenshot proving the renderer mounts is at
`screenshots/reference/00_smoke.jpg` (193 KB, React root present, no
bundler errors).

### Routing caveat

`app.jsx` uses `useState({ kind: 'home' })` and a `go(kind, payload)` helper —
there are **no hash or path URLs**. To navigate the live renderer you must
open the floating "Tweaks" panel and use its `Navigate` section, or fire
clicks programmatically. The exact subcategory / sub-subcategory shown in the
reference PNGs (HLS, DASH, AI/ML Tools) is **not reachable** from the panel —
those PNGs were captured from the production app at `new.awesome.video`, not
from the standalone demo. **Use the PNGs in `uploads/` as the authoritative
reference**; only fall back to live-rendering the bundle when a PNG is
missing or you need a hover/focus state.

## How to view

The handoff folder is best opened directly via `file://` in agent-browser
(see snippet above). A static server on port 7777 was attempted but
Replit's port-watcher only recognizes a fixed port allowlist (3000, 5000,
5173, 8000, etc.), so the workflow was removed. If a downstream task
genuinely needs HTTP serving (e.g. to test relative-path resource loads),
run `python3 -m http.server 8000 --bind 127.0.0.1` from the handoff dir on
one of the supported ports.
