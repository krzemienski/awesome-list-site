## CRITICAL LEARNING (PA-D) — container bakes dist into the IMAGE
- Dockerfile builder stage runs `pnpm build` and copies /app/dist INTO the image.
- Therefore: host `pnpm build` + `docker compose restart app` does NOT update the served bundle. The container keeps serving the OLD baked dist.
- CORRECT fix-deploy cycle for ANY client/server code change: `docker compose up -d --build app` (rebuilds the image), then wait health, then hard-reload the browser.
- Detection trick: `grep -rc "<removed string>" dist/` on host shows 0 (host dist is fresh) yet the browser still shows it => stale container image.
- This applies to EVERY code fix in this run (PA-D dup header, and any later PA-I/PA-* fixes).
