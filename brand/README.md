# awesome.video — Brand Kit

The official identity for awesome.video: the **Inverted Monogram** mark (black rx16 tile,
crimson border, crimson "AV") with the `awesome` + `.video` wordmark, formalizing the
Editorial + Crimson design system used in production.

Everything here is generated by `node scripts/brand/build-brand-assets.mjs`.
Do not hand-edit the outputs — edit the script and regenerate.
All SVGs have text **outlined to paths** (no fonts required by consumers).

## Structure

| Path | Contents |
|---|---|
| `logo/svg/` | Vector masters: `mark.svg` (default, stroke 4), `mark-small.svg` (≤24px, stroke 5), `mark-favicon.svg` (16px, stroke 6), `wordmark*.svg`, `lockup-horizontal*.svg`, `lockup-stacked.svg`. `*-light-bg` variants swap ink `#f4f3ee` → `#0e0d0c`; the tile itself never changes. |

| `social/` | `avatar-512.png` (profile image for GitHub/X/YouTube/LinkedIn), banners + SVG masters for X (1500×500), GitHub (1280×640), YouTube (2560×1440), LinkedIn (1584×396), templates for IG post (1080×1080), IG story (1080×1920), YouTube thumbnail (1280×720), and `og-default.png` (captured from the app's live `/og-image.png` endpoint). |

## Logo rules

- Grid: 76-unit square; tile `x=2 y=2 w=72 h=72 rx=16`, fill `#000000`, stroke `#ff3d52`.
- Monogram: `AV`, Inter 800 at 34/76, letter-spacing −1/34 em, fill `#ff3d52`, baseline y=51.
- Stroke compensation: 4 by default, 5 at ≤24px (`mark-small.svg`), 6 at 16px (`mark-favicon.svg`).
- Clearspace: 0.5× tile width on all sides. Minimum size 16px.
- Wordmark: `awesome` Inter 700 (ink) + `.video` Fraunces italic 600 (crimson), baseline-aligned.
- Never: recolor the AV white, fill the tile crimson, add shadows or gradients, rotate,
  or place on busy imagery without a scrim. On light backgrounds the tile stays black.

## Color

Pure black `#000000` background only. Ink `#f4f3ee` (AAA), muted `#a8a4a0` (AAA),
crimson `#ff3d52` (AA — large text/UI only; use tint `#ffb4be` for small crimson text),
violet `#b84dff` sparingly. Depth comes from hairlines `rgba(244,243,238,0.12)` and
surface `#0e0d0c` — never shadows. See `tokens/tokens.json` for OKLCH values and roles.

## Social template specs

All templates share the same recipe: pure black field, top-left crimson radial glow
(26% → 0 opacity), centered horizontal lockup (≤78% of width), tagline in Fraunces
italic tint `#ffb4be`, tracked +0.28em, auto-fit to ≤88% of width.

| Use | Size | Safe area notes |
|---|---|---|
| X / Twitter header | 1500×500 | Keep lockup inside center 1200×380 (avatar overlaps bottom-left). |
| GitHub social preview | 1280×640 | Full bleed OK. |
| LinkedIn banner | 1584×396 | Profile photo covers bottom-left ~360px — lockup stays centered. |
| YouTube channel art | 2560×1440 | TV-safe center 1546×423 holds the lockup + tagline. |
| YouTube thumbnail | 1280×720 | Replace tagline with episode title (Inter 800, ink) for real videos. |
| IG post | 1080×1080 | Swap tagline for post copy; keep lockup in upper half. |
| IG story | 1080×1920 | Middle third only; top/bottom 250px reserved for UI chrome. |
| OG / link cards | 1200×630 | Generated live by the app (`/og-image.png?path=…`); `og-default.png` is the home-page capture. |

## Voice

Refined, editorial-technical. Sentence case, precise nouns, no hype.
Tagline: *The index for people who ship video.*
Eyebrows: Fraunces italic, uppercase, tracked out, crimson tint.

## Regenerating

```bash
node scripts/brand/build-brand-assets.mjs
```

Fonts (Inter 700/800, Fraunces italic 600 — OFL-licensed static TTFs) are vendored
in `scripts/brand/fonts/` so regenerated output is byte-stable; a Google Fonts
download to `/tmp/brandtools/fonts` is only a fallback if the vendored files are
missing (the script warns loudly, since a hosted-font update could shift glyph
metrics). Note: `social/og-default.png` is **not** produced by the script — it is a
capture of the live `/og-image.png?path=/` endpoint and must be re-captured
separately after any `buildOgSvg` change. Live spec boards for this kit are on the
workspace canvas (Color & Typography, Logo System, Brand In Action, Guidelines & A11y).

## Relationship to the runtime design system

This kit formalizes the **Editorial + Crimson** skin as the canonical awesome.video
brand. The tokens in `tokens/` mirror the runtime values in
`client/src/styles/design-system.css` (`:root[data-system="editorial"]` +
`[data-accent="crimson"]`) — if those change, regenerate this kit to match.
The other 4 runtime skins (Terminal / Geist / Brutalist / Swiss) and 10-accent
switcher are **product theming**, not brand: use only Editorial + Crimson in
external/off-product contexts. The full DS contract lives in
`docs/DESIGN-SYSTEM.md` and `awesome-list-site-ds/HANDOFF.md`; this document
adds brand usage rules on top and does not override either.
