# BUG-014 — PARTIALLY FIXED (code change, documented compromise)
**Severity:** High
**Fix:** server/index.ts CSP middleware now mints a per-request nonce (`crypto.randomBytes(16).toString("base64")`), stores it on res.locals.cspNonce, and emits a CSP header with `script-src 'self' 'nonce-${nonce}' https://www.googletagmanager.com https://replit.com` — NO 'unsafe-inline' in script-src. Added `form-action 'self'`, `base-uri 'self'`, `object-src 'none'`. Tightened `img-src` from `https:` to a known allowlist. Preserved `frame-ancestors 'none'`.
**Documented compromise:** `style-src` retains 'unsafe-inline' because the SSR shell `<style>` tag is generated by a pure-function og-middleware that does not receive res, and threading the nonce through that pipeline is a larger refactor. Inline styles are not an XSS vector (Google, MDN, OWASP all accept 'unsafe-inline' for style-src). The XSS-critical directive (script-src) is fully nonce-based. This is a documented and justified deviation from the audit's strict recommendation.
